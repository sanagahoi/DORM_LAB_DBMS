# Database Management Systems (DBMS) Lab

Welcome to the **DBMS Laboratory** repository! This repository serves as a structured portfolio of database design, schema implementation, and hands-on SQL query exercises completed as part of the Database Management Systems curriculum.

---

## Repository Structure

The repository keeps things clean and straightforward with the following core files:

```text
├── TableCreation&Insertion.sql   # Database schema definitions and initial dataset insertion
├── Queries.sql                  # Comprehensive SQL query exercises (Joins, Aggregations, Subqueries)
└── README.md                    # Repository overview and documentation
```

## Key Database Concepts Covered
This lab focuses on core relational database concepts, implementing them from scratch using structured SQL scripts:

1. Schema Design & Constraints
Table Creation (TableCreation&Insertion.sql):  Defining tables with precise data types, primary keys, foreign keys, and referential integrity constraints.

Data Population: Writing structured INSERT statements to seed the tables with mock data for testing.

2. Advanced Querying (Queries.sql)
Relational Joins: Fetching relational data across multiple tables using Inner, Left, and Outer joins.

Aggregations & Grouping: Utilizing functions like COUNT, SUM, AVG, MAX, and MIN along with GROUP BY and HAVING clauses to extract analytical insights.

Subqueries: Writing nested and correlated queries to handle complex, multi-layered data retrieval tasks.

## Tech Stack & Environment
Language: Structured Query Language (SQL)

Compatible Database Engines: Oracle SQL, MySQL, or PostgreSQL

Recommended Clients: Oracle SQL Developer, DBeaver, DataGrip, or pgAdmin

## How to Use This Repository
Clone the repository:

```
git clone [https://github.com/sanagahoi/DORM_LAB_DBMS.git](https://github.com/sanagahoi/DORM_LAB_DBMS.git)
cd DORM_LAB_DBMS
```
## Set up the Database:

Open your SQL editor/client and connect to your database instance.

Run the entire TableCreation&Insertion.sql script to build the tables and populate them with the initial dataset.

Run the Queries:

Open and execute the queries inside Queries.sql to test and analyze the database outputs.
